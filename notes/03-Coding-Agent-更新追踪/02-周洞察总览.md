# Coding Agent 周洞察总览

本页是 coding agent 竞争追踪的洞察层入口，不追求“把本周所有更新复述一遍”，而是每周只做两件事：

1. 提炼 `1 个最值得研究的设计点`
2. 给出 `1 个最该产出的行动点`

它服务的目标不是新闻消费，而是：

- 逐步建立 coding agent 竞争地图
- 判断哪些能力正在变成行业共性
- 判断哪些设计仍然是某家 agent 的独特点
- 持续把行业洞察转成你自己的 agent 开发能力

## 当前方法

- 检查频率：每周一次
- 证据窗口：最近 `2 周`
- 证据范围：只用官方 source
- 输出风格：`1 个研究点 + 1 个行动点`
- 行动点原则：输出优先，且同时给出 `notes` 产物与 `repo` 内最小验证方向

## 当前基线判断

首个 baseline 下，最近两周最强的共性信号不是“谁又多了一个功能”，而是：

> `runtime control plane` 正在成为 coding agent 的共同竞争面。

这里的 control plane 包括但不限于：

- sub-agent / child-thread 的生命周期
- plan mode / read-only gate
- background jobs / artifacts / resume
- user input / approval / permission surfaces
- provider / model route 与 metadata
- plugin / MCP / remote execution 的接入边界

## 周洞察职责

每周的 `weekly/YYYY-MM-DD.md` 应优先回答：

- 最近两周什么变化最值得研究
- 这件事到底是独特点、行业共性，还是行业收敛
- 对你做 coding agent 的架构和能力路线意味着什么
- 本周最值得留下什么产物
