## OpenAI Codex 今日更新

- 官方 release 列表出现 `0.142.0-alpha.4`
- 发布时间：`2026-06-19 09:39:58Z`
- 直接链接：[0.142.0-alpha.4](https://github.com/openai/codex/releases/tag/rust-v0.142.0-alpha.4)

需要注意的是：这条 alpha release 的正文几乎为空，只有简短的版本标识。  
因此，今天的“新 release”本身是事实，但真正高信息密度的内容仍然来自紧邻的 `0.141.0`，其重点集中在：

- remote executors 的认证与加密 relay
- cross-platform remote execution 的 cwd / shell / permission path 保真
- per-thread plugin / stdio MCP 激活与 marketplace 侧能力
- child threads / external-agent import / rate-limit credits
- request user input 的 auto-resolve timer

也就是说，`Codex` 今天的信号更像是“继续快速迭代”，而不是“今天这条 release 正文本身承载了大量新叙事”。

今天没有确认到新的正式版本头。当前最近已发布版本仍是：

- `0.8.62` - `2026-06-17`

最近一版最重要的点仍然是：

## Reasonix 今日更新

今天没有确认到新的正式 release。当前最近已知 release 是：

- CLI：[`v1.9.1`](https://github.com/esengine/DeepSeek-Reasonix/releases/tag/v1.9.1)
- Desktop：[`desktop-v1.9.1`](https://github.com/esengine/DeepSeek-Reasonix/releases/tag/desktop-v1.9.1)

Reasonix 当前更值得注意的不是某个单一新功能，而是最近 release body 里持续出现的一组“工程化修补点”：

- plan-mode 的只读 gate / bash whitelist
- background job artifacts 的持久化与迁移
- session re-home / stranded session 修复
- no-auth provider / global provider credentials / MCP global config
- detached background jobs、settings refresh lag、scroll / dock / titlebar 等工作台细节

这说明它的重点很偏“工作台、会话、作业、配置和恢复”的产品化稳定性。
