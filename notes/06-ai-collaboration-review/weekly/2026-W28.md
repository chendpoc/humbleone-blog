# AI 协作复盘 · 2026-W28

## 本周一句话判断

本周最重要的改进不是生成第一篇周报，而是把 AI 协作复盘的边界先固定清楚：私有证据可以具体，公开输出必须脱敏；完整分析可以尖锐，发布内容必须可公开、可复用、可长期阅读。

## 1. 决策质量反馈

本周有效的决策动作是把“每周对 work 反馈”重新定义成“AI 协作复盘”。这个重命名很关键，因为它把目标从工作总结转向能力增长：

- 决策质量
- prompt 质量
- workflow 质量
- 对话结构质量
- 学习与成长节奏

需要继续改进的是：当目标同时包含“私有复盘”和“发布到公开网站”时，必须先锁定公开边界。否则后续实现会在隐私和可传播性之间摇摆。

## 2. Prompt 反馈

本周可复用的 prompt 模式是：

```text
请不要只总结我做了什么，而是复盘我如何与 AI 协作：
1. 哪些决策被明确确认，哪些被隐式带过
2. 哪些 prompt 让输出更稳定，哪些造成偏移
3. 哪些 workflow 过重、过轻或需要沉淀
4. 我的对话结构哪里可以提高信息密度
5. 最后只给一个下周最值得练习的动作
```

这个 prompt 的优点是把复盘对象从“任务结果”切到“协作方式”。下次可以进一步要求 AI 标明每条建议的落点：规则、模板、skill、记忆或暂不沉淀。

## 3. Workflow 反馈

当前 workflow 应该保持三段式：

1. 私有证据读取
2. 私有完整复盘
3. 公开脱敏发布

这个顺序不能反过来。先写公开稿容易导致分析变浅；先保留私有完整复盘，才能允许更尖锐、更具体的反馈，再从中提炼适合公开的版本。

## 4. 对话结构反馈

本周对话的有效结构是先表达目标，再给出可选方案，然后选择方案。这里需要额外注意：当一个方案被选择后，如果它和早先目标存在冲突，应该立即重新解释边界。

本周的关键例子是“方案 C”和“发布到 GitHub Pages”之间的张力。正确处理方式不是二选一，而是拆成两层：

- 完整复盘私有
- 公开页面只发布脱敏摘要

## 5. 学习与成长反馈

这套系统最有价值的学习方向不是“让 AI 自动给建议”，而是持续训练自己提出更好的约束：

- 什么材料可以读
- 什么内容不能公开
- 什么建议必须可执行
- 哪些经验值得沉淀
- 下周只练习一个点

长期看，这会把 AI 使用从临时问答推进到个人能力系统。

## 7. 下周一个动作
