# AI 协作复盘总览

这个栏目记录每周一次的 AI 协作复盘。

它不是普通工作周报，也不是对话流水账。它只关注一个问题：

> 我如何更好地使用 AI，形成更清晰的决策、更稳定的 prompt、更可靠的 workflow，以及更可持续的学习节奏？

## 复盘对象

每周优先复盘最近一周的 Codex 本地协作记录，包括：

- 关键任务中的目标表达
- 决策确认方式
- prompt 和约束写法
- workflow 使用是否过轻或过重
- 对话结构是否帮助 AI 更快进入正确问题
- 哪些经验适合沉淀为规则、模板、skill 或记忆

第一版只纳入 Codex 本地历史，不纳入 ChatGPT、Claude Code 或其他 AI 工具导出。

## 公开边界

公开页只发布脱敏后的方法论总结，不发布完整对话历史。

公开内容必须避免：

- 未脱敏的对话材料
- 私有项目细节
- 本地绝对路径
- 密钥、凭证、账号或通知地址
- 未确认可以公开的任务名、客户名或内部计划

更具体的证据、尖锐反馈和原始分析只保存在本地私有工作区。

## 发布方式

这个栏目采用自动发布：每周生成公开脱敏稿后，先运行内容安全检查和站点构建；只有检查通过，才提交公开笔记并触发 GitHub Pages 发布。

如果检查失败，本周复盘会停留在本地，不进入公开网站。

## 固定结构

每篇周复盘使用同一结构：

1. 本周一句话判断
2. 决策质量反馈
3. Prompt 反馈
4. Workflow 反馈
5. 对话结构反馈
6. 学习与成长反馈
7. Workflow Update 候选
8. 下周一个动作

## 质量标准

每篇复盘必须满足三点：

- 至少给出一个可复用的 prompt 或 workflow 改进点
- 每条 Workflow Update 候选都标明建议落点
- 最后只保留一个下周动作，避免变成泛泛建议清单
