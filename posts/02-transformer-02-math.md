## 1. 向量：一堆数字凭什么有意义？

Transformer 里到处是向量。

Token Embedding 是向量，Attention 的 Query / Key / Value 是向量，每一层的 hidden state 也是向量。

那向量到底是什么？

### 1.1 从购物清单到向量

先看一个最简单的例子。

你去超市买三种东西：

| 物品 | 数量 |
| --- | --- |
| 苹果 | 3 个 |
| 牛奶 | 2 瓶 |
| 面包 | 1 袋 |

可以把这张清单写成一个向量：

$$
\mathbf{x} = [3, 2, 1]
$$

这里的 $[3, 2, 1]$ 就是一个三维向量。

- 第一个位置永远是苹果的数量。
- 第二个位置永远是牛奶的数量。
- 第三个位置永远是面包的数量。

这个约定非常重要——向量的每个位置代表一个固定的"含义维度"，同一个维度在不同向量里必须对应同一个含义。

你的清单是 $[3, 2, 1]$，邻居的清单是 $[1, 4, 0]$。两个向量长度一样、维度对齐、含义对应。

在这个三维空间里，每个购物清单就是一个点。

### 1.2 向量 = 空间中的一个点 + 一个方向

一个向量可以两种视角来看：

- **空间视角**：它是高维空间中的一个点。
- **方向视角**：它是从原点指向这个点的一支箭。

![向量在二维空间中的几何表示](/images/transformer-02/vector-2d.png)

以上是二维向量 $\mathbf{v} = [3, 2]$ 的几何表示。它既是一个点 $(3, 2)$，也是一支从原点到这个点的箭头。

这两种视角在 Transformer 里都有用：

- 看"两个 token embedding 的距离"时，向量是空间中的点。
- 看"Attention 里 Q 对 K 的投影方向"时，向量是方向。

### 1.3 在 Transformer 里，向量代表什么？

回到 Transformer。

假设 hidden size 是 4096，那么每个 token 的表示就是一个 4096 维向量。

你没法用纸笔画出来 4096 维空间，但直觉是一样的：

每个 token 在高维空间里占一个位置。语义相近的 token，在高维空间里位置也近。

```text
[0.12, -0.03, 0.87, ..., -0.21]   ← "猫" 的 embedding
[-0.44, 0.19, 0.05, ..., 0.33]   ← "狗" 的 embedding
[0.08, 0.62, -0.31, ..., 0.05]   ← "火锅" 的 embedding
```

4000 多列里，每一列不代表"苹果的数量"或"牛奶的数量"，而是代表某种抽象语义维度。

模型在训练中自己学会了如何分配这些维度：

- 某一列可能偏"生物 vs 非生物"。
- 某一列可能偏"可食用 vs 不可食用"。
- 某一列可能偏"中文语境下的情感倾向"。

这些维度不是人工定义的，是模型在海量文本中自动学出来的。

::: tip 一个帮助记忆的粗糙类比
向量 = 一张包含很多列的性格测试表。

每个 token 在这张表上有自己的分数。分数接近的两个 token，性格接近，在高维空间里位置也接近。
:::

### 1.4 向量加减的直觉

向量可以加减：

$$
\mathbf{a} = [2, 3], \quad \mathbf{b} = [1, -1]
$$

$$
\mathbf{a} + \mathbf{b} = [3, 2]
$$

几何上，向量加法是"一支箭接着另一支箭走"。

放在 Transformer 里，有一个重要应用：**位置编码就是向量加法**。

```text
最终输入 = Token Embedding + Position Encoding
```

也就是把语义向量和位置向量加在一起。

你在第一篇里问过"这不会把语义污染掉吗？"——答案在高维空间里：高维空间足够大，语义维度和位置维度可以基本不打架，模型能学会识别两种模式的叠加。

另一个向量计算是向量减法，它可以用来衡量两个向量的差异：

$$
\mathbf{d} = \mathbf{b} - \mathbf{a}
$$

这个差向量代表从 $\mathbf{a}$ 走到 $\mathbf{b}$ 需要"走几步，往哪个方向走"。

类比一下：北京到上海的位移向量。这个向量不关心你现在在哪儿，只关心从北京出发需要往哪个方向走多远才能到上海。

在 Transformer 里，两个 token embedding 的差向量可以近似表达语义差异。这是一些可解释性研究的基础思路之一。

## 2. 矩阵乘法：批量变换的机器

矩阵就是一张表。

矩阵乘法就是一次"批量变换"。

### 2.1 矩阵是什么？

矩阵就是数字排列成的二维表格：

$$
\mathbf{W} =
\begin{bmatrix}
w_{11} & w_{12} & w_{13} \\
w_{21} & w_{22} & w_{23} \\
w_{31} & w_{32} & w_{33} \\
w_{41} & w_{42} & w_{43}
\end{bmatrix}
$$

这个矩阵是 $4 \times 3$ 的：4 行，3 列。

在 Transformer 里，矩阵通常是**模型参数**。训练学出来的就是这些矩阵里的数字。

### 2.2 矩阵乘法的直觉：线性变换

如果把向量 $\mathbf{x} = [x_1, x_2, x_3]$ 乘以矩阵 $\mathbf{W}$：

$$
\mathbf{y} = \mathbf{x} \mathbf{W}
$$

结果 $\mathbf{y}$ 是一个新向量。它的每个分量，是 $\mathbf{x}$ 各分量的加权和。

展开来看：

$$
y_1 = x_1 w_{11} + x_2 w_{21} + x_3 w_{31}
$$
$$
y_2 = x_1 w_{12} + x_2 w_{22} + x_3 w_{32}
$$
$$
y_3 = x_1 w_{13} + x_2 w_{23} + x_3 w_{33}
$$
$$
y_4 = x_1 w_{14} + x_2 w_{24} + x_3 w_{34}
$$

直觉上，矩阵乘法做了三件事：

1. **维度变换**：输入是 3 维，输出是 4 维。矩阵决定新向量的维度。
2. **信息重组**：输出的每一个分量都是输入所有分量的加权混合。信息被重新搅拌。
3. **变换规则可学习**：矩阵里的数字就是"搅拌规则"。改变规则，搅拌结果就不一样。

### 2.3 Transformer 里矩阵乘法在干什么？

Transformer 里到处都是矩阵乘法。以下是几个核心位置：

**Embedding 查表 → 就是矩阵乘法**

```text
[1, 100000] 的 one-hot × [100000, 4096] 的 Embedding Matrix
→ [1, 4096] 的向量
```

Token ID 本质上是一次"从嵌入矩阵里挑出对应行"的操作。用矩阵乘法来表达，就是拿一个 one-hot 向量乘以嵌入矩阵。

**Q / K / V 投影 → 矩阵乘法**

```text
输入 hidden [seq_len, hidden_size]
× W_Q [hidden_size, head_dim]
→ Query [seq_len, head_dim]
```

输入表示 $\mathbf{X}$ 右乘权重矩阵 $\mathbf{W}_Q$，得到 Query 矩阵。K 和 V 同理。这是 Attention 的入口。

**Attention 计算 → 矩阵乘法**

$$
\text{Attention}(Q, K, V) = \text{softmax}\left(\frac{QK^T}{\sqrt{d_k}}\right) V
$$

这里有两次矩阵乘法：

1. $QK^T$：计算所有 token 对之间的点积（相似度）。
2. 结果矩阵 $\times V$：根据权重聚合信息。

**FFN → 两次矩阵乘法**

```text
FFN(x) = ReLU(xW_1 + b_1)W_2 + b_2
```

隐藏维度先扩张到 $4 \times$，变换后再压回原维度。它本质上是通过两次矩阵乘法完成一次非线性变换：先把信息展开到更高维空间，在那里做变换，再压回原维度。

**LM Head → 矩阵乘法**

```text
最后一个 hidden [1, hidden_size]
× W_LM [hidden_size, vocab_size]
→ logits [1, vocab_size]
```

把最后一个 token 的隐藏状态映射到词表大小，得到每个候选 token 的原始分数。

::: tip 一句话总结
矩阵乘法 = 高维空间的变换器。

Transformer 里几乎所有"把一种东西变成另一种东西"的操作，底层都是矩阵乘法。
:::

### 2.4 矩阵乘法为什么这么快？

CPU 和 GPU 对矩阵乘法有极深的硬件优化。

现代 GPU 的算力很大程度上就是为矩阵乘法设计的。Tensor Core 专门做 $4 \times 4$ 矩阵乘加，一个时钟周期就能完成。

这也是 Transformer 设计中的一个工程约束：

> 如果操作可以被表达成矩阵乘法，那它就能跑得很快。

Self-Attention 里所有 token 同时互相关注，正是因为整个计算被表达成了矩阵乘法，而不是串行循环。

## 3. 点积：为什么乘完加起来就能量相似度？

点积是 Attention 的核心运算。

两个向量做点积：对应位置相乘，然后加起来：

$$
\mathbf{a} \cdot \mathbf{b} = a_1 b_1 + a_2 b_2 + ... + a_n b_n
$$

$$
\mathbf{a} \cdot \mathbf{b} = \sum_{i=1}^{n} a_i b_i
$$

关键问题来了：

> 为什么乘完加起来，就能表示两个向量的相关性？

### 3.1 几何直觉：方向越一致，点积越大

点积有一个等价公式：

$$
\mathbf{a} \cdot \mathbf{b} = \|\mathbf{a}\| \cdot \|\mathbf{b}\| \cdot \cos \theta
$$

其中 $\|\mathbf{a}\|$ 是 $\mathbf{a}$ 的长度（模），$\theta$ 是两支箭之间的夹角。

这意味着：

- 两支箭方向完全一致：$\theta = 0°$，$\cos \theta = 1$，点积最大。
- 两支箭互相垂直：$\theta = 90°$，$\cos \theta = 0$，点积为 0。
- 两支箭方向完全相反：$\theta = 180°$，$\cos \theta = -1$，点积为负且绝对值最大。

![点积的几何含义](/images/transformer-02/dot-product-geometry.png)

所以点积本质上在问：

> 这两支箭，有多"同向"？

同向 → 正数。垂直 → 零。反向 → 负数。

### 3.2 数值直觉：模式匹配

不看几何，只看数值，点积也在做一件事：模式匹配。

假设两个三维向量：

$$
\mathbf{a} = [1, 2, 3]
$$
$$
\mathbf{b} = [1, 2, 3]
$$

它们的点积是：

$$
1\times1 + 2\times2 + 3\times3 = 1+4+9 = 14
$$

换一对：

$$
\mathbf{c} = [1, 2, 3]
$$
$$
\mathbf{d} = [-1, -2, -3]
$$

点积是：

$$
1\times(-1) + 2\times(-2) + 3\times(-3) = -1-4-9 = -14
$$

再换一对（数值起伏完全无关）：

$$
\mathbf{e} = [1, 2, 3]
$$
$$
\mathbf{f} = [2, -5, 0.7]
$$

点积变成一个没什么特别的大小。

规则很简单：

- 两个向量在同样位置上**同时大**或**同时小** → 贡献正值，点积趋向正大。
- 两个向量在同样位置上**一个大一个小** → 贡献负值，点积趋向负大。
- 两个向量在同样位置上**没什么规律** → 正负抵消，点积接近 0。

### 3.3 在 Attention 里，点积在量什么？

Attention 里的点积在量 **Query 和 Key 的匹配程度**。

- Query：当前 token 在问"谁和我相关？"
- Key：每个 token 在说"你需要我的时候，我在。"

点积越大 → Query 和 Key 越匹配 → 这个 token 越值得关注 → Attention 权重越高。

::: tip 直觉类比
你去图书馆找书。

你脑子里有一个关键词列表（Query）。

书架上每本书有一个标签列表（Key）。

你把你的关键词列表和每本书的标签做比对——关键字对上得越多越准，这本书就越值得拿出来。

点积做的是同一件事：比较两个列表在各个维度上是不是同时高、同时低。
:::

而除以 $\sqrt{d_k}$ 是为了做**缩放**，避免向量维度太大导致点积值过大，进而导致 Softmax 梯度过小。这个会在后面 Softmax 一节展开。

## 4. Logits：概率之前的原始分数

模型在最后一层输出时，并不会直接给出概率。

它先输出 logits。

### 4.1 Logits 是什么？

Logits 是**没有经过归一化的原始分数**。

如果词表大小是 100,000，那么最后一个 token 会输出一个 100,000 维的向量，每个位置有一个数。这个向量就是 logits：

```text
logits = [2.3, -1.5, 4.8, 0.1, ..., -3.2]
```

每个数对应一个候选 token 的"原始倾向分数"。

分数高的 token，模型更倾向于选它。分数低的 token，模型更倾向于不选。

但 logits 不是概率，它有几个特点：

1. **可能大于 1 或小于 0**。概率必须在 0 到 1 之间，但 logits 可以是任何实数。
2. **加起来不等于 1**。概率必须加起来等于 1，logits 不需要。
3. **它们之间只有相对意义**。logits = $[0.2, 0.5]$ 和 logits = $[20, 50]$，变成概率后可能是一样的。因为绝对大小会在后续 Softmax 中被消掉。

### 4.2 为什么叫 logits？

这个名字来自统计学里的 logit 函数 $f(p) = \ln(\frac{p}{1-p})$。

logit 函数把 $[0, 1]$ 的概率映射到整个实数轴 $(-\infty, +\infty)$。

虽然现代深度学习里 logits 的数值和这个定义不完全一致，但直觉相似：

> Logits 是概率空间在实数空间里的"展开表示"。

概率被压缩在 $[0, 1]$ 里面，不方便直接做加法、乘法和最优化。logits 把概率展开到整个实数轴，计算就方便了。算完之后，再用 softmax 把结果压回 $[0, 1]$。

### 4.3 Logits → 概率：只差一个 Softmax

这句在下一节展开。这里先记住：

```text
Logits → Softmax → 概率分布
```

Logits 是原材料，Softmax 是加工过程，概率分布是成品。

## 5. Softmax：如何把分数变成概率？

Softmax 是 Transformer 里出现频率最高的函数之一。

它在两个位置出现：

1. Attention 内部：把点积分数变成注意力权重。
2. LM Head 之后：把 logits 变成下一个 token 的概率分布。

### 5.1 从最简单的想法开始

如果给你一堆分数 $[3, 1, 2]$，怎么把它们变成概率？

最简单的想法是归一化：每个数除以总和。

$$
\text{分数} \div \text{总和} = \text{概率}
$$

```text
总和 = 3 + 1 + 2 = 6
概率 = [3/6, 1/6, 2/6] = [0.5, 0.167, 0.333]
```

这个做法直观，但有问题：

**不能有负数。**

如果分数里有负数，比如 $[3, -1, 2]$，总和变成 4，-1 对应 -0.25。但概率不能是负数。

### 5.2 Softmax 的解决方案：先取指数，再归一化

Softmax 的做法是两步：

第一步：对每个分数取指数 $e^x$。这会消掉负数（$e$ 的任意实数次方都是正数）。

第二步：再归一化。

$$
\text{softmax}(x_i) = \frac{e^{x_i}}{\sum_j e^{x_j}}
$$

对 $[3, 1, 2]$ 做 Softmax：

$$
e^3 \approx 20.09
$$
$$
e^1 \approx 2.72
$$
$$
e^2 \approx 7.39
$$

$$
\text{总和} \approx 20.09 + 2.72 + 7.39 = 30.20
$$

$$
\text{结果} \approx [0.665, 0.090, 0.245]
$$

三个性质自动满足：

1. 每个数都在 $(0, 1)$ 之间。
2. 全部加起来等于 1。
3. 相对大小关系保持不变：原始分数高的，概率也高。

### 5.3 Softmax 的温度 (Temperature)

Softmax 有一个可调参数叫温度 $T$：

$$
\text{softmax}(x_i) = \frac{e^{x_i / T}}{\sum_j e^{x_j / T}}
$$

- $T = 1$：标准 Softmax，什么都不变。
- $T \to 0$：概率集中在分数最高的那一项。模型变得"固执"。
- $T \to \infty$：概率趋向均匀分布。模型变得"随意"。

![Softmax temperature 效果](/images/transformer-02/softmax-temperature.png)

在实际使用中，通过调节温度，可以控制模型生成文本的随机性：温度低，输出更确定、更保守；温度高，输出更多样、更有创意。

### 5.4 为什么除以 $\sqrt{d_k}$？

Attention 公式里有一个缩放因子：

$$
\text{Attention}(Q, K, V) = \text{softmax}\left(\frac{QK^T}{\textcolor{red}{\sqrt{d_k}}}\right) V
$$

原因在于：当 $d_k$ 很大时，$Q$ 和 $K$ 的点积值也会变大。点积的方差会随着 $d_k$ 线性增长。

点积太大 → $e^{x_i}$ 很容易爆到非常大的数 → Softmax 的输出集中在某几项 → 梯度接近 0（饱和）。

除以 $\sqrt{d_k}$ 就是为了把点积值的方差控制在合理范围，防止梯度消失。

::: details 补充：为什么变大就会梯度消失？
Softmax 的梯度在概率极端接近 0 或 1 时会接近于 0。

如果点积值太大，Softmax 会把几乎所有概率分配给分数最高的那第一第二项，其余项概率接近 0。

这样大部分 Attention 权重梯度为 0，模型学不动。

缩放等于给 Softmax 加一个"温度调节器"，让概率分布不要过于集中。
:::

## 6. 概率分布：模型的输出到底是什么？

### 6.1 概率分布是什么？

概率分布就是**所有可能结果各自的概率列表**。

掷骰子：

```text
1: 1/6
2: 1/6
3: 1/6
4: 1/6
5: 1/6
6: 1/6
```

两个约束：

1. 每个概率在 $[0, 1]$ 之间。
2. 所有概率加在一起等于 1。

### 6.2 模型输出的概率分布

假设词表只有五个词：

```text
["我", "你", "吃", "火锅", "今天"]
```

模型输出下一个 token 的概率分布可能是：

```text
我   : 0.03
你   : 0.01
吃   : 0.05
火锅 : 0.85
今天 : 0.06
```

"火锅"的概率最高，所以模型最容易选它。

注意："今天"已经出现过（在"我今天想吃"里），但概率分布不关心这个。它只看当前所有候选 token 的相对得分。当然在实际解码时，可以用惩罚、过滤等策略避免重复。

### 6.3 交叉熵：衡量预测分布和真实分布的差距

如果正确答案是"火锅"，那么真实的概率分布是：

```text
我   : 0
你   : 0
吃   : 0
火锅 : 1
今天 : 0
```

这种只有一个位置是 1、其余都是 0 的分布叫 **one-hot 分布**。

模型的预测分布和真实分布之间有一个差距。

Cross Entropy（交叉熵）就是用来量化这个差距的。

这个放在第 9 节详细讲。这里先记住：概率分布是 softmax 的输出，交叉熵是衡量输出好不好的尺子。

## 7. 均值、方差、标准差：LayerNorm 的前置知识

这三个概念是 LayerNorm 的基础。

### 7.1 均值：队伍的平均身高

均值就是所有数值加起来除以个数：

$$
\mu = \frac{1}{n} \sum_{i=1}^{n} x_i
$$

如果一列数是 $[2, 4, 6, 8, 10]$，均值是：

$$
\mu = \frac{2+4+6+8+10}{5} = \frac{30}{5} = 6
$$

均值告诉我们这组数"大概在哪里"。

### 7.2 方差：大家散得多开？

方差衡量这组数"有多分散"：

$$
\sigma^2 = \frac{1}{n} \sum_{i=1}^{n} (x_i - \mu)^2
$$

也就是：

1. 每个数减去均值。
2. 平方。
3. 加起来再平均。

$[2, 4, 6, 8, 10]$ 的方差是：

$$
(-4)^2 + (-2)^2 + 0^2 + 2^2 + 4^2 = 16 + 4 + 0 + 4 + 16 = 40
$$

$$
\sigma^2 = \frac{40}{5} = 8
$$

比较两组数：

```text
A = [1, 2, 3, 4, 5]    方差 = 2
B = [1, 1, 3, 5, 5]    方差 = 3.2
```

A 更紧凑，B 更分散。

方差大 → 数据更散。方差小 → 数据更集中。

在 Transformer 里，每层输出的 hidden state，不同维度之间的方差如果过大或过小，会影响后续层的计算稳定性。

### 7.3 标准差：方差开根号

标准差是方差的平方根：

$$
\sigma = \sqrt{\sigma^2}
$$

它和原始数据同一单位，比方差更直观。

对于 $[2, 4, 6, 8, 10]$：

$$
\sigma = \sqrt{8} \approx 2.83
$$

直观理解：这组数平均在 6 附近，大多数值在 6 ± 2.83 的范围内。

## 8. LayerNorm：为什么要做归一化？

LayerNorm 是 Transformer 里几乎每层都有的组件。

在进入 Attention 和 FFN 之前，数据都会经过 LayerNorm。

### 8.1 问题：数据越传越"散"

Transformer 的 hidden state 在每一层处理之后，数值分布会逐渐偏移：

- 有些维度的值变得非常大（十几个数量级）。
- 有些维度的值变得非常小（接近零）。
- 整体分布偏离初始假设。

这会导致几个后果：

1. 梯度不稳定：有的参数学得飞快，有的参数几乎不动。
2. 模型难收敛：每一层看到的输入分布都不一样，等于每一层都得重新适应。
3. 训练容易崩：数值过大或过小都可能导致梯度过大或过小。

### 8.2 解决方法：归一化

LayerNorm 做的事很简单：

> 对每个 token 的 hidden state 向量，调整它的均值为 0，标准差为 1。

具体步骤：

1. 计算这个向量的均值 $\mu$。
2. 计算这个向量的标准差 $\sigma$。
3. 每个分量都做：

$$
\hat{x}_i = \frac{x_i - \mu}{\sigma}
$$

做完之后，这个向量的均值变成 0，标准差变成 1。

然后再加上两个可学习的参数 $\gamma$（缩放）和 $\beta$（平移）：

$$
y_i = \gamma \hat{x}_i + \beta
$$

$\gamma$ 和 $\beta$ 让模型保留一定灵活性，防止归一化之后把有用的信息也冲掉了。

### 8.3 类比：体重数字和统计表格

你每天称体重：

```text
[60.2, 61.0, 59.5, 62.3, 58.8, ...]
```

这个数列有均值也有方差。如果直接拿这些数字做计算，不同的量级会影响稳定性。

你把它"标准化"：每个数减去均值，再除以标准差：

```text
[-0.3, 0.5, -1.0, 1.8, -1.7, ...]
```

现在均值是 0，标准差是 1。每列都是标准的"偏离单位"，相互之间可以比较。

LayerNorm 做的就是这件事，只不过对象不是体重数字，而是 4096 维的 hidden state 向量。

### 8.4 LayerNorm 跟 BatchNorm 有什么不同？

| | BatchNorm | LayerNorm |
| --- | --- | --- |
| 归一化方向 | 沿 batch 维度（同一特征在不同样本间） | 沿 feature 维度（同一样本的不同特征间） |
| 对 batch size 的依赖 | 依赖，batch 太小效果差 | 不依赖 |
| 训练/推理一致性 | 推理时用全局统计量，行为与训练不同 | 训练和推理完全一致 |
| Transformer 适用 | 不适合序列建模 | 天然适合 |

Transformer 处理的是变长序列，不同的句子里 token 数量不同。而且每个 token 的计算在序列长度维度上本来就需要并行——BatchNorm 在这个场景下很难用。

LayerNorm 对每个 token 单独做归一化，不依赖 batch 里的其他样本，也不依赖序列里的其他 token（至少 Pre-LN 是这样），因此成了 Transformer 的标准选择。

## 9. Cross Entropy：怎么量模型预测得准不准？

第 6 节说了概率分布，现在回到关键问题：

> 训练时，怎么告诉模型"你预测得不对，差距有多大"？

这个"差距"就是损失函数（Loss Function）。

Cross Entropy 是 Transformer 训练中最常用的损失函数。

### 9.1 从直觉出发

如果正确答案是"火锅"：

真实分布：

```text
我: 0, 你: 0, 吃: 0, 火锅: 1, 今天: 0
```

完美的预测分布：

```text
我: 0, 你: 0, 吃: 0, 火锅: 1, 今天: 0
```

这时候 loss 应该是 0——完全正确。

但怎么可能完美呢？实际上：

预测分布（之一）：

```text
我: 0.03, 你: 0.01, 吃: 0.05, 火锅: 0.85, 今天: 0.06
```

真实答案是"火锅"，模型分给"火锅"的概率是 0.85。

如果模型分给正确答案的概率越高，loss 越小。如果概率越低，loss 越大。

Cross Entropy 就是量化这个逻辑的数学工具。

### 9.2 公式

对于分类问题，真实分布 $y$ 是 one-hot（只有一个正确答案），Cross Entropy 退化成：

$$
\text{Loss} = -\ln(p_{\text{correct}})
$$

其中 $p_{\text{correct}}$ 是模型分配给正确答案的那个概率。

- $p_{\text{correct}} \to 1$：$-\ln(1) = 0$，loss 很小。
- $p_{\text{correct}} \to 0$：$-\ln(0) \to +\infty$，loss 很大。

![Cross Entropy 曲线](/images/transformer-02/cross-entropy-curve.png)

这是一个非常好的"惩罚曲线"：越接近正确答案，奖励越陡；越远离正确答案，惩罚越重。

### 9.3 为什么不直接用"正确概率"而是取对数？

直接最小化 $1 - p_{\text{correct}}$（也就是"错误概率"）不好吗？

不好。因为对数有放大效应。

假设正确答案的概率从 0.01 提升到 0.02，用"错误概率"看，loss 从 0.99 降到 0.98，变化只有 0.01，梯度信号很弱。

用 Cross Entropy 看，loss 从 $-\ln(0.01) \approx 4.6$ 降到 $-\ln(0.02) \approx 3.9$，变化有 0.7，梯度信号更足。

所以取对数之后，模型在概率很低时的改进能得到更强的梯度信号，学习更快。

### 9.4 在 Transformer 训练中的应用

GPT 训练每一批数据时做的事大约是：

1. 给模型前 $n-1$ 个 token。
2. 模型预测第 $n$ 个 token 的概率分布。
3. 用真实第 $n$ 个 token 的 id 和预测分布计算 Cross Entropy loss。
4. 反过来更新模型参数（梯度下降，见下一节）。

一句话总结：

> Cross Entropy 衡量"模型对正确答案有多确信"。越不信就罚得越重。

## 10. 偏导数与梯度下降：模型怎么从错误中学习？

这是 Transformer 数学链条的最后一块拼图。

### 10.1 问题：怎么让 loss 变小？

训练模型是一个优化问题：

> 调整模型参数，让 Cross Entropy Loss 尽可能小。

但模型有几亿甚至几千亿个参数，没法手动调。需要自动方法。

### 10.2 偏导数：单独拧一个旋钮的效果

想象模型只有一个参数 $w$。Loss 是 $w$ 的函数 $L(w)$。

导数 $\frac{dL}{dw}$ 告诉你：在当前位置，$w$ 往正方向微调一点，loss 会变大还是变小。

- 导数为正 → $w$ 增大 → loss 增大 → 应该减小 $w$。
- 导数为负 → $w$ 增大 → loss 减小 → 应该增大 $w$。

对于多个参数，偏导数就是"单独拧这一个旋钮，其他不动，看 loss 怎么变"。

所有参数的偏导数放在一起，就是**梯度** $\nabla L$。

### 10.3 梯度下降：沿着最陡的下坡走

梯度是一个向量，指向"loss 上升最快的方向"。

那要减少 loss，就往反方向走：

$$
w_{\text{new}} = w_{\text{old}} - \eta \cdot \nabla L
$$

![梯度下降示意图](/images/transformer-02/gradient-descent.png)

其中 $\eta$ 是学习率（Learning Rate），控制每一步迈多大。

- 学习率太大 → 可能越过最低点，来回震荡，甚至 diverges。
- 学习率太小 → 收敛太慢，训练几十个 epoch 还没到位。
- 学习率合适 → 平稳下降，逐步接近最优。

直觉类比：

> 你在浓雾中站在一座山上，想走到山脚。你看不到整座山，但能感觉到脚下往哪个方向最陡。你就一小步一小步地往最陡的下坡方向走。

这就是梯度下降。

### 10.4 反向传播：怎么算出每个参数的偏导数？

Transformer 有几十层，几百亿参数，显然不能一个一个参数去试。

反向传播（Backpropagation）是一个高效计算所有偏导数的算法。

它的核心是链式法则：

> 如果 $z = f(y)$, $y = g(x)$，那么 $z$ 对 $x$ 的导数 $\frac{dz}{dx} = \frac{dz}{dy} \cdot \frac{dy}{dx}$。

在神经网络里：

1. **前向传播**：从输入层往输出层计算，得到 loss。
2. **反向传播**：从 loss 往回传梯度，用链式法则逐层计算每个参数的偏导数。
3. **参数更新**：用梯度下降更新参数。

反向传播不是本书的重点，这里只给直觉：

> 模型在前向传播时记住每一步的计算路径，反向传播时沿着路径倒着走，把梯度一根一根传回去。

这就像多米诺骨牌倒序推演：最后一块牌倒的方向和力度取决于前面每一块牌的摆放。沿着链条倒回去，你就能算出每一块牌的位置应该怎么调。

### 10.5 梯度下降的变体

实践中很少用最朴素的梯度下降。常见变体：

- **SGD（随机梯度下降）**：每次只用一小批数据算梯度，而不是全量数据。更快，但有噪声。
- **Adam**：自适应学习率。每个参数有自己的学习速度。大多数 Transformer 训练的默认选择。
- **AdamW**：Adam + 权重衰减（Weight Decay）。在 Adam 的基础上把权重衰减从 L2 正则化里拆出来，效果更好。也是 LLaMA 等模型常用的优化器。

::: details 为什么用 Adam 而不是朴素 SGD？
Transformer 参数非常多，不同层的梯度量级差异很大。

朴素 SGD 用一个全局学习率走，没法照顾到这种差异。Adam 给每个参数维护自己的动量和平滑历史，能自适应当前参数的梯度特性。

但也因此 Adam 比 SGD 更"激进"，有时会在验证集上比 SGD 泛化能力稍差。工程上 Adam/AdamW 已经是主流选择。
:::

## 12. 总结

这篇文章的目标不是教你做数学题，而是给每一个公式配一幅画面。

核心收获清单：

1. **向量 = 高维空间中的点 + 方向**。Embedding 就是给每个 token 在高维空间里找一个位置。
2. **矩阵乘法 = 线性变换机器**。Transformer 里几乎所有变换都是矩阵乘法。
3. **点积 = 方向一致性检测器**。Attention 用点积量 Query 和 Key 的匹配度。
4. **Logits = 概率之前的原始分数**。还没有归一化，可以是任何实数。
5. **Softmax = 分数转概率的机器**。先取指数消负数，再归一化。
6. **概率分布 = 所有候选结果各自的概率**。加起来等于 1。
7. **均值、方差、标准差** 是 LayerNorm 的前置概念。均值是"中心在哪"，方差是"散得多开"。
8. **LayerNorm = 把每个向量的均值调成 0、标准差调成 1**。让深层网络训练稳定。
9. **Cross Entropy = 衡量预测分布和真实分布的差距**。惩罚"模型给正确答案的概率不够高"。
10. **偏导数 + 梯度下降 = 模型从错误中学习的方式**。沿着最陡的下坡一步步调整参数。

如果能用自己的话解释下面几个问题，这篇的目标就基本达成了：

8. Cross Entropy 的 $-\ln(p)$ 公式在表达什么？为什么取对数？
